# Project Rules

- **HANDOFF.md Maintenance:** You MUST keep the `HANDOFF.md` file in the root directory up to date at all times. Whenever you implement new features, change architectural logic, fix significant bugs, or release new versions (App or Firmware), you must automatically update `HANDOFF.md` to reflect these changes before concluding your task. Do not ask for permission to do this, simply treat it as part of your standard definition of done.
- **Release & Push Automation:** Whenever you make changes to the **Windows App (C#)**, you MUST automatically push the changes to Git and create a new release by bumping the app version, creating a Git tag (e.g. pp-vX.Y.Z), and pushing it. Similarly, whenever you make changes to the **Firmware (C++/PlatformIO)**, you MUST push the changes and create a new firmware release by bumping the firmware version, creating a Git tag (e.g. w-vX.Y.Z), and pushing it. Do not ask for permission to do this, treat it as a standard part of finishing your task.
